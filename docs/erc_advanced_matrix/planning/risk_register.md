# Risk Register

| Risk | Impact | Mitigation |
| --- | --- | --- |
| MATRIX is perceived as software integration rather than frontier science. | Very high | Frame around molecular-state theory, predictive spectroscopy and new observables; keep code details secondary. |
| Scope is too broad for one ERC project. | High | Use five WPs with clear dependencies; choose 3-5 demonstrators; defer nano-LEGO/TRINITY production details. |
| Unified molecular state becomes too rigid for diverse chemistry. | High | Version schemas, allow adapter-specific metadata, validate on diverse benchmark classes. |
| Normal-mode mapping fails for difficult multi-level models. | Medium | Use overlap diagnostics, fallback to local-mode/GIC mapping, report uncertainty. |
| VSCF/vibronic/IP-EA extensions overrun. | Medium | Stage as WP4 with minimal viable protocols before full GUI workflows. |
| Experimental database comparisons are inconsistent. | Medium | Save provenance, phase/state, query URL, units and selection filters. |
| Lump-sum budget is underjustified. | High | Tie costs to deliverables and personnel tasks from the start. |
| Host-institution internal deadline is earlier than expected. | High | Confirm immediately and upload a near-final version early. |
